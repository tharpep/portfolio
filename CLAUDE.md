# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm dev        # Start dev server with Turbopack
pnpm build      # Production build
pnpm lint       # ESLint
```

No test suite is configured.

## Architecture

**Stack:** Next.js 15 App Router, React 19, TypeScript, Tailwind CSS v4, pnpm. Deployed on Vercel. Branch `dev` is the working branch; `main` is production.

**Path alias:** `@/*` → `src/*`

### Pages

| Route | File |
|---|---|
| `/` | `src/app/page.tsx` |
| `/about` | `src/app/about/page.tsx` |
| `/projects` + `/projects/[slug]` | `src/app/projects/` |
| `/resume` | `src/app/resume/page.tsx` |
| `/photography` + `/photography/[slug]` | `src/app/photography/` |

### Data sources

**Projects** — All project data lives as a static array in `src/lib/getProjects.ts`. To add/edit a project, edit that file. Set `hidden: true` to exclude a project from all listings without deleting it.

**Photography** — Collections are auto-discovered from Cloudinary folders under `photography/`. No hardcoded list. Just upload a folder to Cloudinary and it appears automatically. Manual overrides (camera, location, `featured`) live in `collectionOverrides` in `src/lib/collections.ts`. Collection metadata (title, description, mood) is generated by Claude Haiku (`claude-haiku-4-5-20251001`) via `src/lib/ai-metadata.ts` and cached 24h with `unstable_cache`.

**Spotify** — `src/data/spotify-data.json` is a static file updated daily by the GitHub Actions workflow (`.github/workflows/spotify-sync.yml`), which runs `spotify-scripts/fetch_spotify_data_github.py`. The Next.js app reads this file directly via `src/lib/spotify-data.ts`.

### Navigation / Layout

The root layout (`src/app/layout.tsx`) renders `<ConditionalNav>` and `<ConditionalFooter>`, which suppress themselves on any `/photography` route. The photography section has its own `<PhotoNav>` component and its own layout at `src/app/photography/layout.tsx`.

### Environment variables

| Variable | Used by |
|---|---|
| `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME` | Cloudinary (public) |
| `CLOUDINARY_API_KEY` | Cloudinary server calls |
| `CLOUDINARY_API_SECRET` | Cloudinary server calls |
| `ANTHROPIC_API_KEY` | Photography collection metadata via Claude Haiku |

Spotify secrets (`SPOTIFY_CLIENT_ID`, `SPOTIFY_CLIENT_SECRET`, `SPOTIFY_REFRESH_TOKEN`) are only needed for the GitHub Actions workflow, not for running the Next.js app locally.
